🧠 L1 — Purpose (High-Level Summary)
Write 1–2 sentences describing what this code is supposed to do.Example:“This function ingests raw NFL stats and transforms them into a clean feature matrix for training a regression model.”

🔁 L2 — Flow Summary (Step-by-Step Logic)
Use short bullet points to describe what the code does from top to bottom.
* Step 1:
* Step 2:
* Step 3:
* Step 4:
* Step 5:

🛠 L3 — Key Mechanisms / Patterns Used
Explain why the code works and what engineering or algorithmic patterns are inside.
Use bullets like:
* Uses a ___ pattern (e.g., memoization, recursion, factory pattern)
* Leverages vectorized operations for performance
* Uses type hints for clarity
* Applies async concurrency to avoid blocking
* Uses dependency injection for modularity
* Implements a BFS-like traversal
* Applies sliding-window technique
* Uses batching for GPU efficiency
This is where you build intuition.

⚠️ L4 — Edge Cases / Weak Points / Risks
List the areas where the code might fail or behave unexpectedly.
Consider:
* Input edge cases
* Missing values
* API response failures
* Performance bottlenecks
* Security risks
* Threading/concurrency issues
* Off-by-one errors
* Assumptions made by the AI
Be honest here — this builds debugging skill.

✍️ L5 — My Rewrite / Modification
Rewrite ANY part of the code — even if it’s small.
Examples:
* Rename variables
* Change control flow
* Extract helper function
* Improve readability
* Add comments
* Add type hints
* Optimize a loop
This makes the pattern part of your long-term memory.

📚 L6 — Meta (What I Learned / Where Else This Applies)
Reflect for 2–3 sentences.
Prompts:
* “What concept did I learn here?”
* “Where else can this pattern be used?”
* “How would a senior engineer explain this?”
* “What questions should I ask AI about this next time?”
This step consolidates long-term intuition.





For every AI-generated block of code, ask:
* What problem is this solving?
* What are the inputs/outputs?
* What assumptions is it making?
* What’s the time/space complexity?
* Where can this break?

Even if everything is perfect, change:
* variable names
* function names
* ordering of operations
* minor logic changes
* comments

Even if it works, break it intentionally:
* remove imports
* change values
* force an error
* modify an API call

“Give 3 different implementations.”
Compare:
* readability
* performance
* architecture
* modularity

Force yourself to write 2–4 sentences:
- “This function does X by doing Y. The key idea is Z. It relies on A/B/C.”

Ask AI to quiz you:
* “Give me 5 questions about this code.”
* “Hide the code, ask me to recreate it.”
* “Ask me where the edge cases are.”

You will regress only if you:
* Stop reading the code
* Stop understanding why it works
* Never write or fix code yourself
* Treat AI output as magic
* Skip debugging
* Don’t build real projects
